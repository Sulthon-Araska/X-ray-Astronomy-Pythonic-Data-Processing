# **X-ray Astronomy Pythonic Data Processing Pipelines**
A collection of non-official Python-based data processing pipelines for ([HEASARC](https://heasarc.gsfc.nasa.gov/)-listed) X-ray astronomical observation missions. The goal of these pipelines is to streamline the process of accessing, calibrating, cleaning, extracting, correcting, and merging raw observational data into high-level scientific data products (images, light curves, and spectra) for each observation.

**_Disclaimer!_**
This repository is still in early development. _Currently_, the developer has no rigid milestone for continous development and might only update it when the necessity comes. Therefore, any improvements are highly welcomed!


## **General Descriptions**
1. Each Jupyter notebook (IPYNB) file is dedicated to one X-ray mission. Each mission may have more than one instrument and may support multiple observation mode (see https://heasarc.gsfc.nasa.gov/docs/observatories.html and their respective official page for more details). The pipeline codes here, however, may not _yet_ support all the instruments and/or modes available. **Users are obligated to check the compatibility between what the pipeline supports and their scientific needs.**
2. The pipeline code heavily utilizes astronomy-oriented Python packages (and its dependencies), such as [``astropy``](https://www.astropy.org/) (for basic astronomical functions and data manipulation), [``Stingray``](https://docs.stingray.science/) (for advanced light curve corrections), and [``heasoftpy``](https://heasarc.gsfc.nasa.gov/docs/software/lheasoft/heasoftpy/) (official Python interface to [HEASoft](https://heasarc.gsfc.nasa.gov/docs/software/lheasoft/) [FTOOLS](https://heasarc.gsfc.nasa.gov/docs/software/ftools/ftools_menu.html); for high-level mission-specific data processing). Users may need to install these packages and HEASoft to their working space beforehand.
3. The pipeline code were originally created in [SciServer](https://apps.sciserver.org/) cloud environment using ``HEASARCv6.36`` image container. Thus, data accessing and files management are _currently_ hardcoded for SciServer storage environment. Usage in SciServer (either interactive or asynchronous) cloud computing should works well. Nevertheless, **users are still implored to reasses and modify the downloaded source code to suit their own working space and scientific goals.**
4. Although the goal of the pipeline is to ease the needs of data processing, users are **strongly advised to understand the workings** behind each step of the pipeline (_please read the corresponding package/software's documentations!_) in order to be able to correctly customize the code when in need.
